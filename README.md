## Work I did on this project?
For this project I started with the provided Jupyter Notebook starter code that set up the treasure maze environment and the overall structure for training. My main work was filling in and adjusting the deep Q learning pieces so the pirate agent could actually learn instead of just speed running into walls like it was doing a challenge run. I worked on the training loop behavior like choosing actions using exploration vs exploitation, updating epsilon, and tracking performance like win rate so I could tell if it was improving or just getting lucky.

## What computer scientists do and why it matters?

Computer scientists build systems and algorithms that solve problems across basically every field, from apps people use daily to security tools that keep data from getting yoinked. It matters because good software saves time, reduces errors, and can literally affect safety and money in the real world, which is kind of a big deal. Even in AI and machine learning, the goal is not magic, it is building tools that can make smarter decisions with real constraints and real consequences.

## How I approach problems as a computer scientist?

One reason I like computer science so much is I am super hands on and I like making things. The difference is with computers I get to build stuff that is still real in a useful way, but it does not take up physical space and it does not require buying materials every time I want to try something. I already paid for the computer anyway, so the upfront cost is basically done, and after that it is mostly just electricity and my time.
I also like that mistakes are not permanent the way they can be in woodworking. If you mess up a nice piece of walnut while making a chair, that mistake is forever and you are either changing the plan or crying a little inside. With software, if I mess up, I can revert to a snapshot or roll back changes and get right back to a clean state. Because of that, my approach is to break each problem into smaller problems, test each piece to see how it behaves, then combine them into a bigger solution. I try to build those smaller pieces in a way I can reuse later, so I am not reinventing the wheel every time, I am just stacking better wheels into a working car.

## Ethical responsibilities to the end user and the organization?

 My ethical responsibility is to build systems that are secure, reliable, and honest about their limits, especially when AI is involved because people love to trust outputs that sound confident. For end users that means protecting privacy, avoiding biased behavior, and not pretending the system is smarter than it is. For the organization it means protecting data and intellectual property, following policies and laws, and not shipping something that turns into a security incident later. Basically I want my work to help people, not end up as the reason someone adds a new rule to the employee handbook.
